## Short and Long Read Simulation

This repository goes over the methodology for validating the SNVs, Indels, and SVs within our somatic variant benchmarkset using an alignment based approach. See READMEs in subdirectories for more information.

## Assembly Based Validation

This repository goes over the methodology for validating SNVs, Indels, and SVs using an assembly based approach. See READMEs in subdirectories for more information.

## Additional Resources

`order_sensitivity_pad100.bed.gz`

This file defines genomic regions where somatic variant calls are sensitive to pangenome graph construction order. 

It was generated by extracting all false positive (FP) and false negative (FN) variants from pairwise benchmarking comparisons across three Minigraph-Cactus pangenome graphs built from identical inputs in different sample insertion orders:

- Order 1: HG005 -> HG00438 -> HG002 -> HG02257 -> HG02486 -> HG02622
- Order 2: HG005 -> HG02622 -> HG002 -> HG02257 -> HG02486 -> HG00438
- Order 3: HG00438 -> HG002 -> HG02257 -> HG02486 -> HG02622 -> HG005

All variant types (SNVs, indels, SVs) were included. Each variant was converted to a genomic interval using its REF allele span (0-based: start = POS-1, end = POS-1 + len(REF)), then padded by 100 bp on each side and merged with overlapping windows using bedtools. *The final file contains 173,249 merged intervals covering 39.4 Mb.* These regions are strongly enriched in tandem repeats (8.67x) and abnormal GC content (2.01x). 

We highly recommend using sequence-context-based variant comparison tools / in-house scripts for comparing and validating variants falling within these regions, as variant representation is proven to differ depending on the reference genome used and the first haplotype added during graph construction process.
